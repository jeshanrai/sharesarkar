# sharesanskar

Monorepo with a Next.js frontend (`frontend/`) and an Express + Postgres backend (`backend/`).

## Local development

Backend:

```bash
cd backend
cp .env.example .env   # fill in DB_* values
npm install
npm run seed           # creates schema + demo admin (admin / admin123)
npm run dev
```

Frontend:

```bash
cd frontend
cp .env.example .env.local   # set NEXT_PUBLIC_API_URL=http://localhost:5000
npm install
npm run dev
```

## Deployment

The project is configured for **Vercel** (frontend), **Render** (backend), and **Neon** (Postgres).

### 1. Database — Neon

1. Create a project on [neon.tech](https://neon.tech).
2. Copy the pooled connection string (it ends with `?sslmode=require`).
3. From your local machine, bootstrap the schema and admin user against Neon:
   ```bash
   cd backend
   DATABASE_URL="postgresql://...neon.tech/...?sslmode=require" npm run seed
   ```
   Change the default admin password (`admin / admin123`) immediately after first login.

### 2. Backend — Render

Render reads [`render.yaml`](render.yaml) at the repo root.

1. Push this repo to GitHub.
2. In Render → **New → Blueprint**, point at the repo. It will create the `sharesanskar-backend` web service.
3. Set the env vars (the blueprint marks them `sync: false`):
   - `DATABASE_URL` — the Neon connection string from step 1.
   - `FRONTEND_URL` — comma-separated list of allowed origins, e.g. `https://sharesanskar.vercel.app`. Leave empty to allow all (not recommended in prod).
   - `JWT_SECRET` is auto-generated by Render.
4. Deploy. Health check is `/api/health`.

### 3. Frontend — Vercel

1. In Vercel → **Add New → Project**, import the same repo.
2. Set **Root Directory** to `frontend`. Vercel auto-detects Next.js.
3. Add env var `NEXT_PUBLIC_API_URL` = your Render service URL (e.g. `https://sharesanskar-backend.onrender.com`).
4. Deploy. Once live, copy the Vercel domain back into Render's `FRONTEND_URL` and redeploy the backend.

### Notes

- The pg pool enables SSL automatically when `DATABASE_URL` is set, which is what Neon requires.
- The backend runs an in-process scheduler (NEPSE refresh). Render's free tier sleeps after inactivity — upgrade to a paid plan if you need the scheduler running 24/7.
- `backend/.env` and `frontend/.env*.local` are gitignored. Never commit real secrets.
